### async & await in python

async 和 await 是 Python 实现异步编程的核心关键字，专门用来写协程，解决等待耗时操作时不阻塞程序的问题（比如等待网络请求、文件读写、数据库查询）。

在梳理 async 和 await 之前，我们需要先理解几组概念。

第一个是同步和异步的区别：

1. 同步是一件事做完才能做下一件，等待期间程序阻塞、什么都不做。

2. 异步是遇到等待时，先去执行其他任务，等结果返回再继续，**非阻塞**。异步不会让单个任务变快，而是让**多个任务的等待时间重叠**，整体更快。 异步快，不是因为干得快，而是因为不浪费等待的时间，多个任务等待时间重叠 = 总时间大幅缩短。一个任务的完整生命周期：

   > 启动任务 → 【等待 IO / 网络 / 数据库】 → 拿到结果

   “开启任务 → 拿到结果” 之间的时间，就是等待时间。异步就是让这些等待时间重叠，不排队。等待 IO / 网络 / 数据库 的处理，完全不由我们当前的 Python 线程管，全部是操作系统硬件 + 驱动在后台悄悄处理！我们的线程根本不参与、不等待、不干活，只负责等待结果。

也就是同步和异步是描述单个任务执行的时，要不要阻塞、要不要等。

第二个是并发与并行的区别：

1. 并发单核 CPU 即可实现，多个任务**快速交替执行**，同一时刻只执行一个，靠任务切换实现 “看起来同时运行”，适合 IO 密集型（网络、数据库、文件）。
2. 并行指的是有多个 CPU 核心，任务 A 在核心 1 跑，任务 B 在核心 2 跑，同一时刻真的同时做，适合大量计算（压缩、加密、图像处理）。并行也就是我们通常说的多进程。

一句话简单理解就是：并发是交替处理多个任务，并行是真的同时处理多个任务。

需要注意的是同步不等于并发，同步可以是串行（一个接一个），也可以并发（多个任务交替，但每个任务内部是同步阻塞）。

异步也不等于并发，异步只是非阻塞，不一定同时跑多个任务。

但**异步 + 多任务 = 并发**。

不管同步还是异步，只要多核同时跑，就是并行。

协程是可以手动暂停、再恢复执行的函数，也叫用户态轻量级线程。

特点：

- 极轻量，一个线程可开上万协程
- 切换由程序控制（协作式调度），不由操作系统抢占
- 无线程切换开销，基本不需要锁
- 单核即可实现高并发

Python 中 `async/await` 就是**协程的语法**，实现**异步并发**。

async 用在函数定义前，把普通函数变成异步函数（协程函数），需要注意异步函数不能直接调用，直接调用会返回一个协程对象。

```python
# 普通函数
def normal_func():
    return 1

# 异步函数（协程函数）
async def async_func():
    return 1
```

await 指的是等待异步操作完成，只能用在异步函数内部，作用是暂停当前协程，去执行其他任务，等这个操作完成后再回来，后面必须跟可等待对象（异步函数、协程、异步 IO 操作等）。

```python
async def demo():
    # 等待异步函数执行完，再往下走
    result = await async_func()
    print(result)
```

下面是一个最标准的异步代码写法：

```python
import asyncio
import time

# 定义异步函数：模拟耗时操作（比如网络请求）
async def task(name, delay):
    print(f"任务 {name} 开始，需要等待 {delay} 秒")
    # await 暂停当前任务，不阻塞程序
    await asyncio.sleep(delay)  
    print(f"任务 {name} 完成")
    return f"任务 {name} 结果"

# 主异步函数
async def main():
    start = time.time()
    
    # 创建多个异步任务（同时执行，不是串行等待）
    t1 = asyncio.create_task(task("A", 2))
    t2 = asyncio.create_task(task("B", 3))
    
    # 等待所有任务完成
    res1 = await t1
    res2 = await t2
    
    print(res1, res2)
    print(f"总耗时：{time.time() - start:.2f} 秒")

# 运行异步主函数（Python 3.7+ 写法）
asyncio.run(main())
```

执行流程：

1. `asyncio.run(main())` 启动**事件循环**，开始执行异步任务。
2. `create_task` 将协程包装为任务，并**立刻在后台开始运行**。
3. 任务 A 执行 → 打印开始 → 遇到 `await` → 暂停 A。
4. 事件循环切换 → 任务 B 执行 → 打印开始 → 遇到 `await` → 暂停 B。
5. 两个任务**同时等待**，CPU 空闲。
6. 2 秒到 → A 恢复执行 → 完成。
7. `await t1` 完成，继续执行 `await t2`，再等 1 秒。
8. 3 秒到 → B 完成。
9. 总耗时约 **3 秒**，而非 2+3=5 秒。

两个任务是同时开始、同时等待的，总时间不是 2+3=5 秒，而是 取最大的 3 秒，await 不阻塞整个程序，只是暂停当前任务，让别的任务先跑。

需要明白，这两个任务同时运行，绝对不是两个 CPU 核心在并行计算，全程都在同一个进程、同一个线程里执行！不是多核帮忙，而是快速切换！

流程是这样的（**同一个线程**）：

1. 跑 A 任务 → 遇到 `await` → **暂停 A**
2. 立刻切换 → 跑 B 任务 → 遇到 `await` → **暂停 B**
3. 等着... （CPU 闲着）
4. A 等待结束 → **切回 A** 跑完
5. B 等待结束 → **切回 B** 跑完

只是切换快，看起来像。

为什么要这么做？
因为它不耗 CPU，专门用来等：

- 网络请求

- 数据库读写

- 睡眠等待

- 文件读写

CPU 大部分时间在摸鱼，利用等待时间切换任务，效率极高。

总结一下：

1. asyncio 是单线程、单进程
2. 没有用到多核，不是真正的并行
3. 所谓 “同时运行” = 协作式并发 + 任务切换
4. await 就是切换任务的开关

一个线程，能同时开启成百上千个异步任务！全程只有一个线程在来回切换！但能同时处理 N 多任务！协程超级轻、不占新线程、不占新进程、切换成本几乎为 0。这就是异步高并发的秘密。

一个线程，就像一个人，可以同时烧水、煮饭、洗衣服，靠来回切换，同时处理 N 多任务！

多进程编程：多个人（线程）同时干活，同一时刻 可以真正同时运行（如果有多核），线程之间是 操作系统调度切换，会有 GIL 问题（Python 多线程不能同时跑 CPU 密集型）。

单线程异步编程：只有 一个线程，同一时刻 只能运行一个任务任务切换 由程序员控制（await）,无锁、无竞争、超级轻量。

并发是能同时处理多个任务，但不一定同时执行。

只要满足：

- 有多个任务
- 它们看起来像是在 “一起推进”
- 任务之间来回切换

就叫**并发**。

它不关心你是：

- 一个线程
- 多个线程
- 多进程
- 异步

只要多个任务交替执行、共同推进，就是并发。

单线程异步编程就是在一个线程里，用协程 + 事件循环实现并发。

单线程异步、多线程、多线程 + 异步，全都能实现高并发。但高并发的 “成本” 和 “上限” 完全不一样，现实中：单线程异步 > 多线程 > 多线程 + 异步（混合）。高并发的瓶颈不是 “执行速度”，而是 “等待时能不能不占资源”。

单线程异步 = 最高效、最主流、最常用；

多线程 = 简单但笨重；

多线程异步 = 天花板最高，但复杂。
